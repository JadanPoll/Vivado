Vitis XSCT / XSDB Commands (Fully Annotated Tree)
├── Workspace & Project Management (Classic XSCT)
│   ├── app (Create, configure, build, and clean application projects)
│   ├── bsp (Create, configure, and build Board Support Packages)
│   ├── platform (Create and manage hardware platforms from .xsa)
│   ├── sysproj (Create and manage full system projects)
│   ├── repo (Set or get custom IP/BSP repository paths)
│   ├── setws / getws (Set or get the current Vitis workspace directory)
│   └── importprojects (Import existing Vitis projects into the workspace)
├── Hardware Software Interface (HSI)
│   ├── hsi open_hw_design (Open an exported .xsa / .hdf hardware design)
│   ├── hsi get_cells (Get hardware IP blocks in the design)
│   ├── hsi get_pins (Get interface pins on a specific IP)
│   ├── hsi generate_bsp (Generate a BSP directly from the hardware design)
│   └── hsi close_hw_design (Close the currently loaded hardware design)
├── Target Connection & Management (XSDB)
│   ├── connect (Connect to a hw_server or specific target board)
│   ├── disconnect (Disconnect from the current target/server)
│   ├── targets (List, filter, and select active JTAG/TCF targets like APU/RPU)
│   └── jtag (Low-level JTAG commands: set frequency, terminal, shift data)
├── Device Programming & Download (XSDB)
│   ├── device program (Program a specific device over JTAG)
│   ├── device status (Read device status and boot registers)
│   ├── fpga (Download a .bit or .pdi bitstream file to the FPGA/Versal)
│   └── dow (Download a compiled .elf file or raw data to target memory)
├── Debugging & Execution (XSDB)
│   ├── con (Continue execution on the active target processor)
│   ├── stop (Halt execution on the active target processor)
│   ├── rst (Reset the system, a specific core, or the processor)
│   ├── bpadd (Add a hardware/software breakpoint or watchpoint)
│   ├── bpremove / bpdisable (Remove or disable existing breakpoints)
│   ├── bplist / bpstatus (List breakpoints and check if they were hit)
│   ├── n / s (Step over [next] / Step into C/C++ source code)
│   ├── ni / si (Step over instruction / Step into Assembly instruction)
│   └── bt (Print the backtrace of the current thread/process)
├── Registers & Memory (XSDB)
│   ├── mrd (Memory Read: Read bytes/words from a specific hex address)
│   ├── mwr (Memory Write: Write bytes/words to a specific hex address)
│   ├── rrd (Register Read: Read processor/core registers like R0, PC, SP)
│   ├── rwr (Register Write: Write a value to processor/core registers)
│   ├── getaddrmap (Get the memory map of the active target processor)
│   └── mask_write (Perform a safe read-modify-write on a memory address)
└── Profiling & RTOS (XSDB)
    ├── tcfprof (Configure and run TCF-based performance profiling)
    ├── profile (Start/stop software performance profiling)
    ├── getos (Get Real-Time Operating System info for the target)
    └── getdrivers (Get driver information for IP connected to the target)
