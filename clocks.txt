═══════════════════════════════════════════════════════════════════════════════
  CONSTRAINT REFERENCE — WHAT IS AND IS NOT LEGAL
═══════════════════════════════════════════════════════════════════════════════

  BUFG  ──drives──▶  any FF on entire device           ✓ LEGAL
  BUFG  ──drives──▶  any BRAM/DSP clock input          ✓ LEGAL
  BUFR  ──drives──▶  FFs in same clock region           ✓ LEGAL
  BUFR  ──drives──▶  FFs in different clock region      ✗ ILLEGAL (DRC)
  BUFIO ──drives──▶  ILOGIC/OLOGIC in same bank         ✓ LEGAL
  BUFIO ──drives──▶  CLB FFs                            ✗ ILLEGAL
  MMCM  ──drives──▶  fabric directly (no BUFG)          ✗ ILLEGAL / bad skew
  >12 BUFGs active in one region simultaneously         ✗ DRC: CLOCK-012
  CLK routed through INT fabric (no BUFG)               ✗ ns-level skew

  BUFGCE.CE changes asynchronously                      ✗ GLITCH risk
  BUFGCE.CE changes synchronous, clock low               ✓ GLITCH-FREE

       └─────────────▶┌─────────┐

    on this crossing   └────┬────┘
                            │
                            ▼
                       safe data
                       in clk_B domain

  MTBF formula:
  MTBF = (T_CLK / (f_data × T_W)) × e^(T_resolve / τ)
  
  At 200MHz, τ=35ps, T_W=60ps, f_data=100MHz:
  T_resolve = 5ns - 1ns = 4ns
  MTBF ≈ 2.9 × 10^49 seconds  (effectively infinite)
  → Two-FF synchronizer is universally sufficient


═══════════════════════════════════════════════════════════════════════════════
  STARTUP SEQUENCE (power-on clock behavior)
═══════════════════════════════════════════════════════════════════════════════

  1. VCCAUX powers up (1.8V)                ← must be first or simultaneous
  2. VCCINT + VCCBRAM power up (1.0V)
  3. VCCO banks power up (1.2-3.3V)
  4. Configuration loads (CCLK active, CFGMCLK ~65MHz available via STARTUPE2)
  5. MMCM/PLL starts, VCO ramps up
  6. LOCKED asserts  ← ONLY now is output clock frequency correct
  7. Release reset to all downstream logic
  8. EOS (End of Startup) via STARTUPE2
  9. Normal operation

  !! Assert reset on ALL logic clocked by MMCM/PLL outputs until LOCKED = 1 !!
  !! Before LOCKED: output frequency is undefined — could be 0, partial, or glitching !!


═══════════════════════════════════════════════════════════════════════════════
  DELAY BUDGET — END TO END
═══════════════════════════════════════════════════════════════════════════════

  Stage                                         Typical delay
  ─────────────────────────────────────────────────────────────
  Crystal pad → IBUF                            0.5 – 1.0 ns
  IBUF → MMCM CLKIN1                            0.1 – 0.3 ns
  MMCM internal (CLKIN → CLKOUT, locked)        phase defined
  MMCM CLKOUT → BUFG input                      0.1 – 0.3 ns
  BUFG input → H-Tree root                      0.2 – 0.5 ns
  H-Tree root → HROW                            balanced, ≈equal per region
  HROW → Clock Spine                            0.1 – 0.2 ns
  Clock Spine → FF clock pin                    0.05 – 0.2 ns
  ─────────────────────────────────────────────────────────────
  Chip-wide skew (FF to FF, same BUFG)          < 200 ps
  Clock uncertainty used by Vivado STA          ~150 ps (jitter + skew margin)
  ─────────────────────────────────────────────────────────────
  BUFR regional distribution skew              < 500 ps (within one region)
  BUFIO I/O-only skew                           < 100 ps (optimized for SERDES)


═══════════════════════════════════════════════════════════════════════════════
  RESOURCE COUNTS — QUICK REFERENCE
═══════════════════════════════════════════════════════════════════════════════

  Global clock buffers (BUFG):     32 total, max 12 active per region
  Regional buffers (BUFR):         6 (one per clock region, integer ÷ 1-8)
  I/O clock buffers (BUFIO):       6 (one per I/O bank, feeds ILOGIC/OLOGIC)
  Multi-region buffers (BUFMR):    6 (drives BUFRs across adjacent regions)
  MMCMs:                           5 (fractional ÷, dynamic phase, spread-spectrum)
  PLLs:                            5 (lower jitter, integer ÷ only, 6 outputs)
  HROWs (Horizontal Clock Rows):   3 (one per row of clock regions)
  Clock regions:                   6 (2 columns × 3 rows)
  VCO frequency range:             600 – 1600 MHz (hard constraint — violating
                                   causes unlock or unreliable output)
